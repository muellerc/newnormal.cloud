A Blog Only Based On S3

December 6, 2016

This blog post shows how to create a simple blog only based on AWS S3.

I'm using Bootstrap's Blog template to jump start. You can find all the required resources in my GitHub repository. Navigate to your AWS console for AWS S3 and start with creating a new bucket of your choice.

NOTE: all bucket names has to be unique across a region and all users!

After you have created the bucket (newnormal.cloud in this example), set up the following folder and file structure (you can find the files in my GitHub repository):

              
newnormal.cloud                   (bucket)
 |- index.xhtml                   (file/object)
 |- js                            (folder)
 | |- bootstrap-3.3.7.min.js      (file/object)
 | \- jquery-1.12.4.min.js        (file/object)
 |- css                           (folder)
 | |- bootstrap.min.css           (file/object)
 | \- blog.css                    (file/object)
 \- images                        (folder)
   |- image.jpg                   (file/object)
   |- enable_website_hosting.png  (file/object)
   |- make_public.png             (file/object)
   \- favicon.ico                 (file/object)

After you have uploaded all files, select you bucket in the console so that the bucket properties are listet as shown below. Unfold the section Static Website Hosting and select Enable website hosting. Enter index.xhtml into the field Index Document and save the changes:

Last but not least, you have to make your files and folders public, so that everybody can view it. To do so, select all files and folder in the root of your bucket and choose Make Public from the Actions menue:

Now you are done and ready to go!
